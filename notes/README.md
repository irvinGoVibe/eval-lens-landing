# Notes

← [[Home]]

`notes/` — слой сырого капчура. Сюда попадает всё, что ещё не дозрело до `wiki/`.

## Структура

```
notes/
  README.md
  daily/   — YYYY-MM-DD.md, потоковая хронология (append-only, с таймстемпами)
  done/    — YYYY-MM-DD.md, структурированный дневной отчёт (один файл на день)
```

## Типы сигналов

| Сигнал | Куда | Префикс `/note` | Linear (если подключён) |
|---|---|---|---|
| Баг, замеченный по ходу работы | `daily/` | `bug:` | Issue в — (после подтверждения) |
| Задача, которую нужно сделать | `daily/` | `task:` | Issue в — |
| Идея для discovery | `daily/` | `idea:` | Issue в — |
| Наблюдение, контекст, сырая мысль | `daily/` | — | нет |
| Что сделано за день | `done/` | — (через `/done`) | нет |

## daily/ vs done/

- **`daily/`** — потоковый capture по мере возникновения. Формат — буллеты `- HH:MM — <текст>`. Через `/note` или вручную.
- **`done/`** — отчёт в конце дня по шаблону `done/_template.md`: сделано / в процессе / блокеры / ссылки. Через `/done` или вручную любым AI-ассистентом.

## Правила

- **Append-only.** Файл сегодняшнего дня дополняется в течение дня; прошлые дни не переписываются.
- **Быстрота важнее формата.** Задача слоя — зафиксировать, а не оформить.
- **Ссылки.** Если заметка относится к документу или задаче — ставится ссылка на `wiki/` или идентификатор Linear issue.
- Заметка живёт, пока её смысл не перейдёт в `wiki/` (через `/curate`); после переноса остаётся как исторический слой.
