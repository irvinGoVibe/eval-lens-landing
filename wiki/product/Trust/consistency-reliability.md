# Consistency & Reliability

## Page purpose

Explain how EvalLense checks whether evaluation results are stable, repeatable, and decision-ready.

## User task

Help the user understand that EvalLense is not only producing scores, but also making reliability visible.

## Main idea to communicate

A score is only useful if you know how stable it is. EvalLense helps reveal consistency, variance, and cases that need human attention.

## What the page should prove

```text
EvalLense cares about evaluation quality, not just output volume.
```

## Section structure

### 1. Hero

Purpose:

```text
Position reliability as the quality layer of evaluation.
```

Message:

```text
Understand where scores are stable, where reviewers disagree, and where human review matters most.
```

### 2. Why reliability matters

Purpose:

```text
Explain the danger of treating every score as equally reliable.
```

Pain points:

```text
Some decks are clear winners or losers
Some decks create disagreement
Some scores may look precise but hide uncertainty
Decision teams need to know where to focus attention
```

### 3. Variance analysis

Purpose:

```text
Explain score spread and disagreement.
```

Message:

```text
Variance shows where evaluation results are aligned or where different judge lenses produce meaningful disagreement.
```

### 4. Repeatability checks

Purpose:

```text
Explain stability across runs or evaluation passes.
```

Message:

```text
Repeatable results increase confidence. Unstable results signal that a case needs closer review.
```

### 5. Outlier detection

Purpose:

```text
Show how unusual judge outputs or scoring patterns can be surfaced.
```

Message:

```text
Outliers are not automatically wrong, but they should be visible to the review team.
```

### 6. Confidence checks

Purpose:

```text
Explain the link between evidence strength and confidence.
```

Message:

```text
When evidence is strong, confidence can be higher. When evidence is weak or unclear, the team should know.
```

### 7. Human escalation

Purpose:

```text
Tie reliability to the human-in-the-loop process.
```

Message:

```text
Reliability signals help humans focus on borderline, inconsistent, or high-impact decisions.
```
