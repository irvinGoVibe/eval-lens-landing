# Entry Hub

## Page purpose

Explain how EvalLense collects applications and pitch decks into one controlled intake flow.

## User task

Help program owners, funds, accelerators, and competition organizers understand how they can create a single entry point for participants.

## Main idea to communicate

Entry Hub replaces scattered intake across email, forms, spreadsheets, and messengers with one controlled submission flow.

## Problem

```text
Pitch decks and participant data often arrive through multiple channels:
- email
- Google Forms
- Airtable
- Notion
- Telegram
- manual uploads
```

This creates messy tracking, missing files, duplicate entries, and slow review preparation.

## What the page should prove

```text
EvalLense gives every batch a clean entry point.
All decks and participant information enter the same review workspace.
```

### 3. How Entry Hub works

Purpose:

```text
Explain the basic user flow.
```

Flow:

```text
1. Create a batch / program
2. Configure the submission page
3. Share the entry link
4. Participants upload pitch decks
5. Submissions appear in the private workspace
6. Evaluation can start from a clean batch
```
