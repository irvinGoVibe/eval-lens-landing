## Контент по секциям

### 1. Hero
- **Заголовок:** A scientific methodology for evaluating pitch decks
- **Подзаголовок:** EvalLense оценивает деки по явным критериям через AI-жюри —
  с прозрачным обоснованием и человеком в контуре финального решения. *(черновик)*
- **CTA:** Book a Demo

### 2. Принципы оценки
Три принципа методологии (из [[design-system|Design System]] §Принципы):
- **Criteria-based evaluation** — оценка по заранее определённым критериям, а не
  по общему впечатлению.
- **Evidence over opinion** — показываем, что нашли, откуда и чего не хватает;
  никакой «голой» оценки без обоснования.
- **AI prepares, humans decide** — вывод AI рекомендательный (advisory); финальное
  решение всегда за человеком.

### 3. Конвейер оценки
Дек проходит фиксированный конвейер (узлы зажигаются по мере обработки):
`Decode → Judges → Summarize → Score → Report`. Каждый шаг детерминирован и
воспроизводим — одинаковый дек проходит один и тот же путь. *(черновик; уточнить
описание каждого шага)*

### 4. Роли судей
Оценку ведёт **AI-жюри из нескольких судей**, каждый отвечает за свой набор
критериев (в продукте — критерии **P1–P6**). Несколько независимых судей снижают
зависимость оценки от одного взгляда. *(черновик; публичная расшифровка P1–P6 —
см. «Открытые вопросы»)*

### 5. Scoring model
Каждый score **привязан к evidence**: breakdown по критериям, цитата-источник со
ссылкой на страницу дека, явные флаги «missing evidence». Score сопровождается
**confidence ring** (уровень уверенности). *(черновик; формула агрегирования —
см. «Открытые вопросы»)*

### 6. Rubric system
Единая рубрика задаёт критерии и уровни, по которым судьи выставляют оценки —
это и делает результат сравнимым между деками. *(черновик; конкретные уровни
рубрики не зафиксированы)*

### 7. Human-in-the-loop
AI выдаёт **advisory**-вывод, который передаётся человеку с named-ролью; человек
ставит финальное решение, audit-строка фиксирует кто и когда. AI-green
зарезервирован под одобрение человека — подчёркивает, кто главный.
