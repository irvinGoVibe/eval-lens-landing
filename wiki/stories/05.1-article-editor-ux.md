# Story 05.1 - Article Editor UX (тулбар, вставка по курсору, drag/paste-в-текст, слэш-меню, табы)

Status: Done (locked)
Owner:
Team: —
Priority:
Linear Mapping: Project by default

> **UX-доработка редактора тела статьи поверх Story 05.** Story 05 перевела CMS на дизайн-систему, локализовала интерфейс на английский и завела нативный drag-and-drop (`ImageDropzone`). Но сам блок редактирования тела статьи (`MarkdownEditor`) остался примитивным: голая Markdown-`<textarea>` 50/50 с превью и единственная загрузка, которая **дописывает `![](url)` в конец тела** — не туда, где курсор. Редактор «выглядит ужасно, неудобно пользоваться». Story 05.1 переделывает именно поверхность редактирования по лучшим практикам Markdown-CMS, **не меняя** канон тела (Markdown), публичный рендер (`MarkdownBody`), директивы (`::video`, `:::gallery`), схему данных и server-side upload. **Story 05.1 не реализуется без Story 02 и 05.**

## Summary

- **Что меняем.** Обогащаем `MarkdownEditor` (тело статьи) до удобной редакторской поверхности, оставляя Markdown каноном: (1) **тулбар форматирования** (bold/italic/inline-code, H2/H3, quote, bullet/ordered list, link, divider) с вставкой по выделению/курсору и горячими клавишами; (2) **вставка медиа по месту курсора** — drag-and-drop файла прямо в текст и paste из буфера вставляют `![](url)` / `::video{src=}` туда, где каретка, с inline-плейсхолдером загрузки (оптимистично), а не в конец тела; (3) **слэш-меню** (`/` в начале строки) для структурных блоков и директив (Heading, Quote, List, Image, Video, Gallery, Divider) — чтобы `::video`/`:::gallery` были обнаружимы без знания синтаксиса; (4) **табы Write / Preview** (+ Split на широком экране со синхро-скроллом) вместо тесного 50/50; (5) **автосейв черновика** в localStorage + предупреждение о несохранённых изменениях; (6) **футер-мета** (слова, символы, оценка времени чтения).
- **Зачем.** Текущий редактор тела заставляет писать Markdown «вслепую», знать синтаксис директив наизусть, а после загрузки картинки — вручную вырезать `![](url)` из конца тела и вставлять по месту. Это медленно и раздражает. Сплит 50/50 не сворачивается, на узком экране тесно.
- **Эффект.** Редактор статьи ощущается как продуктовый инструмент: форматирование в один клик/хоткей, картинка падает прямо в текст, директивы продукта обнаружимы через `/`-меню, черновик не теряется. Канон (Markdown), публичный рендер, директивы, схема и server-side upload — без изменений; публичный блог без регрессий.

## Problem

- **Загрузка не по месту.** `MarkdownEditor` грузит файл через `uploadArticleMediaAction`, но результат `setValue((v) => v + snippet)` **дописывает `![](url)`/`::video{src=}` в конец тела**, игнорируя позицию курсора. Чтобы поставить картинку в нужный абзац, редактор вынужден вручную перемещать сгенерированный Markdown — главный источник «неудобно».
- **Нет форматирования.** Тело — `<textarea className="admin-textarea">` без тулбара и хоткеев. Заголовки, цитаты, списки, ссылки, жирный — всё руками в Markdown. Структурные директивы (`::video`, `:::gallery`) живут только в крошечной строке-подсказке `hint` — не обнаружимы.
- **Тесная раскладка.** `.admin-md` — жёсткий `grid-template-columns: 1fr 1fr` без сворачивания; на узких экранах и textarea, и превью зажаты. Нет режима «только письмо» / «только превью».
- **Нет защиты от потери.** Никакого автосейва/черновика: случайный уход со страницы или перезагрузка теряет несохранённое тело.

## Goal

После Story 05.1 должно стать правдой:

- В редакторе тела статьи есть тулбар форматирования; действия применяются к выделению/курсору (обтекание выделения, префикс строк, вставка), с горячими клавишами (Cmd/Ctrl-B, -I, -K).
- Загруженный/вставленный/перетащенный медиа-файл попадает Markdown'ом **в позицию курсора** (или точку дропа), а не в конец тела; во время загрузки на месте стоит inline-плейсхолдер, заменяемый финальным URL по успеху или удаляемый с понятной ошибкой по сбою.
- Структурные блоки и директивы продукта (`::video`, `:::gallery`, heading/quote/list/divider) доступны через слэш-меню `/` — без знания синтаксиса.
- Редактор предлагает табы Write / Preview (и Split на широком экране); на узком — не тесно.
- Черновик тела автосейвится локально и восстанавливается; уход с несохранёнными изменениями предупреждает.
- Markdown остаётся единственным каноном тела; публичный `MarkdownBody`, директивы и `.article-*` рендер не меняются; upload остаётся через server action (секреты server-side); новых тяжёлых зависимостей нет.

## Scope

### In scope

- **Тулбар форматирования** над textarea тела (`MarkdownEditor`): bold, italic, inline-code, H2, H3, quote, bullet list, ordered list, link, horizontal rule. Кнопки — ghost icon-buttons дизайн-системы (hairline-разделители, violet active). Действия реализуются маленькой чистой утилитой над textarea-selection (`wrapSelection` / `prefixLines` / `insertAtCursor`), восстанавливающей выделение после правки. Хоткеи: Cmd/Ctrl-B (bold), -I (italic), -K (link: оборачивает выделение в `[sel](url)` и ставит каретку в `url`).
- **Вставка медиа по курсору** — переписать поведение загрузки: вместо `value + snippet` вставлять `![](url)` / `::video{src=}` в текущую позицию каретки (а при drop — по позиции дропа, если доступна). Во время аплоада вставляется уникальный inline-плейсхолдер (например `![uploading…#token]()`); по успеху он заменяется на финальный snippet, по ошибке — удаляется, сообщение об ошибке показывается. Замена идёт по уникальному токену, чтобы правки пользователя во время загрузки не сломали позицию.
- **Drag-and-drop в textarea + paste из буфера** — бросок файла прямо на область письма и `paste` изображения из буфера запускают тот же upload-по-курсору. Существующая кнопка-дропзона (`ImageDropzone`) сохраняется как явная аффорданс, но тоже вставляет по курсору (а не в конец).
- **Слэш-меню** — `/` в начале строки (или после пробела/переноса) открывает клавиатурно-навигируемое меню блоков: Heading (H2/H3), Quote, Bullet list, Ordered list, Image (открыть выбор файла), Video (`::video`), Gallery (`:::gallery`), Divider. Выбор вставляет корректный snippet по курсору. Esc закрывает; меню не триггерится, когда `/` — легитимная часть текста (не в начале строки).
- **Табы Write / Preview (+ Split)** — заменить жёсткий 50/50 на сегментед-контрол: на узком экране дефолт Write, переключение в Preview; на широком (≥ ~1100px) доступен Split со синхро-скроллом. Превью остаётся на белом `.article-*` canvas (инвариант — не наследует admin-тему).
- **Автосейв черновика** — тело автосейвится в localStorage по ключу (slug или `new`), восстанавливается при повторном открытии формы (с возможностью «restore / discard», если расходится с серверным значением); навигация с несохранёнными изменениями предупреждает (`beforeunload` / клиентский guard). Черновик чистится после успешного сохранения.
- **Футер-мета редактора** — счётчики слов/символов и оценка времени чтения (информативно, поле `read_minutes` остаётся ручным, как в Story 02).
- **Доступность и фолбэк** — кнопки тулбара — настоящие `<button>` с `aria-label`; слэш-меню и табы навигируются с клавиатуры; DnD/paste — прогрессивное улучшение поверх рабочей `<textarea>` (без JS тело всё равно редактируется); `prefers-reduced-motion` уважается.
- **Стили в `admin.css`** — новые классы (`.admin-mdtoolbar`, `.admin-mdtabs`, `.admin-slashmenu`, плейсхолдер-состояние) строятся только на токенах дизайн-системы (violet primary, hairline, `--r-*`, `--sh-*`, `--ease`); новый «диалект» цветов/теней не вводится.
- **Патч документации** — отметить в `cms.md` обновлённый редактор тела (тулбар, вставка по курсору, слэш-меню, табы, автосейв-черновик) как часть визуально-интерфейсной политики CMS.

### Out of scope

- **Полноценный WYSIWYG / block-движок** (TipTap, Lexical, ProseMirror) и двусторонняя сериализация в Markdown — решение принято в пользу Enhanced-Markdown без тяжёлых зависимостей (см. Open Questions Story 05). Markdown остаётся каноном.
- **Медиа-библиотека / браузер бакета** — листинг/переиспользование/удаление объектов Storage из UI (как и в Story 05 — только upload-в-форме).
- **Версионирование/история ревизий** статьи на сервере — local-draft не равен серверной истории; ревизии вне scope.
- **Обработка изображений** — кроп, ресайз, сжатие, alt-text-редактор за пределами простого ручного ввода в Markdown.
- **Редактор `LoopForm` (репосты)** — Story 05.1 касается тела статьи; file-upload репостов остаётся как в Story 05.
- **Изменение Markdown-рендера, директив, парсера** (`MarkdownBody`, `::video`, `:::gallery`, `.article-*`) — поведение и синтаксис не меняются; редактор лишь генерирует эти же конструкции.
- **Схема данных, миграции, новые поля** — данные Story 01/02 не трогаются.
- **Изменение порта dev (3005), правила ScrollOrchestrator, публичный блог** — без изменений.

## Users / Actors

- primary user: редактор контента — пишет тело статьи; теперь форматирует в один клик/хоткей, бросает картинку прямо в нужный абзац, вызывает блоки через `/`, не боится потерять черновик.
- system actor: Next.js server (server action `uploadArticleMediaAction` под `service_role`, server-only); Supabase Storage (бакет `media`); клиентский `MarkdownEditor` (textarea-модель, тулбар, слэш-меню, автосейв в localStorage).

## Main Flow

1. Подтверждён подход (Enhanced Markdown без тяжёлого WYSIWYG) — зафиксирован в этом диалоге.
2. В `MarkdownEditor` вводится модель работы с textarea-selection (чистая утилита: `wrapSelection`, `prefixLines`, `insertAtCursor`, `replaceToken`), управляющая ref'ом на `<textarea>` и состоянием `value`.
3. Над textarea добавляется тулбар форматирования (ghost icon-buttons) + горячие клавиши; каждое действие применяется по выделению/курсору и восстанавливает selection.
4. Переписывается загрузка: drag-and-drop на textarea и paste из буфера, а также кнопка `ImageDropzone`, вставляют медиа **по курсору/точке дропа** через inline-плейсхолдер → замена по уникальному токену на финальный snippet (успех) или удаление + ошибка (сбой). Upload по-прежнему идёт через `uploadArticleMediaAction` (server action).
5. Добавляется слэш-меню `/` для блоков/директив (Heading/Quote/List/Image/Video/Gallery/Divider) с клавиатурной навигацией и Esc.
6. Раскладка переводится на табы Write / Preview (+ Split на широком экране, синхро-скролл); превью остаётся на `.article-*` белом canvas.
7. Подключается автосейв черновика в localStorage (ключ slug/new), restore/discard при расхождении с сервером, guard на уход с несохранёнными изменениями; чистка после успешного save.
8. Добавляется футер-мета (слова/символы/время чтения).
9. Новые стили (`.admin-mdtoolbar`/`.admin-mdtabs`/`.admin-slashmenu`/плейсхолдер) — на токенах дизайн-системы.
10. Проверяется server-side инвариант (upload через server action; секреты не в клиентском бандле), консистентность превью `.article-*`, отсутствие регрессий публичного блога.
11. Патчится `cms.md` (обновлённый редактор тела).
12. Проверка статикой/диффом; `cd web && pnpm build` — **только по явной просьбе user**.

## Business Rules

- **Markdown — единственный канон тела.** Редактор лишь генерирует Markdown/директивы; `body` сохраняется как Markdown-строка. Никакого скрытого формата, никакой потери при round-trip.
- **Вставка медиа — всегда по курсору** (или точке дропа), никогда не «в конец тела». Во время загрузки — inline-плейсхолдер с уникальным токеном; замена по токену, устойчивая к параллельным правкам.
- **Upload — только через server action** (`uploadArticleMediaAction`, `service_role`, server-only). DnD/paste/слэш-Image — клиентские триггеры, но сам upload и секреты в клиентский бандл не попадают (инвариант CLAUDE.md).
- **Директивы и парсер не меняются.** Слэш-меню и тулбар вставляют ровно те же `![](url)`, `::video{src=}`, `:::gallery`, заголовки/списки/цитаты, что понимает публичный `MarkdownBody`.
- **Превью консистентно публичному рендеру** — на `.article-*` белом canvas, не наследует admin-тему.
- **Прогрессивное улучшение и фолбэк.** Без JS / при сбое улучшений тело редактируется обычной `<textarea>`. DnD/paste — поверх, не вместо.
- **Стили — только токены дизайн-системы**; новый локальный «диалект» цветов/теней/радиусов не вводится. Тулбар/меню — ghost/violet, hairline, `--r-*`, `--sh-*`, `--ease`.
- **Local-draft ≠ серверная истина.** Автосейв в localStorage — защита от потери, не версионирование; при расхождении пользователь явно выбирает restore/discard.
- **Без новых тяжёлых зависимостей.** Реализация — нативная (textarea + небольшие утилиты). Любой пакет — только по согласованию; pnpm-only, `package-lock.json` в `web/` не создавать.
- **Scope — только тело статьи.** Редактор репостов (`LoopForm`), схема данных, порт dev (3005), ScrollOrchestrator — без изменений.

## Edge Cases

- **Drop/paste, когда textarea не в фокусе** — вставка идёт в последнюю известную позицию каретки; если её нет — в конец (зафиксированный фолбэк), но не молча в начало.
- **Правка текста во время загрузки** — позиция плейсхолдера сместилась: замена идёт по уникальному токену, а не по числовому offset, чтобы не перезатереть чужой текст.
- **Сбой/отмена загрузки** — inline-плейсхолдер удаляется, тело не остаётся с «висящим» `uploading…`; форма не теряет введённые данные; ошибка (тип/размер/сбой) показывается понятно.
- **Несколько файлов в drop** — для тела зафиксировать поведение (последовательная вставка по одному snippet'у на файл); тип/размер каждого валидируется (как в `ImageDropzone`: image/video, ≤ 50 MB).
- **Слэш-меню vs легитимный `/`** — меню триггерится только в начале строки/после пробела; Esc и клик вне закрывают; ввод текста после `/` фильтрует пункты, не ломая обычный ввод.
- **Хоткеи vs браузер** — Cmd/Ctrl-B/-I/-K перехватываются только при фокусе в textarea, `preventDefault` без поломки нативного поведения вне редактора.
- **Restore черновика конфликтует с серверным телом** — при открытии формы с непустым серверным `body` и существующим локальным черновиком предложить restore/discard, не перезатирая молча.
- **Markdown-превью после смены раскладки/табов** — `.article-*` превью не наследует admin-цвета, остаётся консистентным публичному рендеру; Split синхро-скролл не «дёргает» при reduced-motion.
- **Очень длинное тело** — textarea и превью скроллятся независимо; автосейв троттлится, чтобы не писать в localStorage на каждый кейстрок.
- **Приватные/служебные символы в URL** — сгенерированный `![](url)`/`::video{src=}` корректно экранирует/оставляет URL как есть (как сейчас), не ломая Markdown.

Возможна небольшая правка `ArticleForm.tsx` (обёртка/лейбл) и переиспользование/расширение `ImageDropzone.tsx` (вставка по курсору вместо append). ## Acceptance Criteria

- [ ] **Given** форма статьи с фокусом в теле, **when** жму кнопки тулбара или хоткеи (Cmd/Ctrl-B, -I, -K) с выделенным текстом, **then** выделение оборачивается/префиксуется корректным Markdown (bold/italic/link/H2/H3/quote/list/code/divider), а каретка/выделение восстанавливаются осмысленно.
- [ ] **Given** курсор в середине тела, **when** перетаскиваю файл в область письма / вставляю изображение из буфера / выбираю файл в дропзоне, **then** во время загрузки на месте курсора стоит inline-плейсхолдер, а по успеху он заменяется на `![](url)` (или `::video{src=}`) **в этой позиции**, не в конце тела; по ошибке плейсхолдер удаляется и показывается понятное сообщение.
- [ ] **Given** правлю текст во время загрузки, **when** загрузка завершается, **then** финальный snippet встаёт на место своего плейсхолдера (замена по уникальному токену), не перезатирая введённый между делом текст.
- [ ] **Given** начало строки, **when** ввожу `/`, **then** открывается клавиатурно-навигируемое слэш-меню (Heading/Quote/List/Image/Video/Gallery/Divider); выбор вставляет корректный snippet/директиву по курсору; Esc закрывает; `/` в середине текста меню не открывает.
- [ ] **Given** редактор тела, **when** переключаю табы, **then** доступны Write и Preview (на широком экране — Split со синхро-скроллом); на узком экране не тесно; превью рендерится на публичных `.article-*` и визуально консистентно публичному рендеру.
- [ ] **Given** несохранённое тело, **when** перезагружаю/повторно открываю форму, **then** черновик восстанавливается из localStorage (с restore/discard при расхождении с серверным `body`); уход с несохранёнными изменениями предупреждает; после успешного Save черновик очищается.
- [ ] **Given** клиентский бандл, **when** грепаю исходники/сборку admin-зоны, **then** `SUPABASE_SERVICE_ROLE_KEY`, `CMS_PASSWORD`, admin-клиент Supabase отсутствуют в клиентском коде; upload идёт через server action, не напрямую из клиента.

## Resolved Decisions

Зафиксировано в диалоге (2026-06-16) — все Open Questions закрыты:

- **Слэш-меню — в MVP.** Полноценное `/`-меню (Heading/Quote/List/Image/Video/Gallery/Divider) входит в первый объём наравне с тулбаром и вставкой-по-курсору: единственный способ сделать директивы `::video`/`:::gallery` обнаружимыми без знания синтаксиса; редактор сразу ощущается цельным.
- **Раскладка — табы + Split на широком экране.** На узком — табы Write/Preview (дефолт Write). На широком (≥ ~1100px) доступен третий режим Split с пропорциональным синхро-скроллом; синхро-скролл гасится при `prefers-reduced-motion`.
- **Restore-конфликт — ненавязчивый баннер.** При расхождении локального черновика с серверным `body` сверху редактора показывается тонкая полоска «Found an unsaved draft — Restore / Discard»; по умолчанию отображается серверное тело, пока пользователь не нажал Restore. Без блокирующей модалки.
- **Троттлинг автосейва — debounce ~800 мс**, ключ `evallens:draft:article:<slug|new>`, очистка после успешного Save.
