# Page Design Patterns — Apple-grade

← [[index|Wiki]] · [[design-system|Design System]]

Как собирать **внутренние страницы**, чтобы они читались как Apple product pages:
ритмичные, разнокалиберные, со scroll-driven появлением, а не плоская сетка
карточек. Это рабочий справочник для skill `build-pages` и `developer-agent`.
Дополняет [[design-system|Design System]] (токены, классы) — здесь про
**композицию, ритм и движение**.

> Правило №1: движение — только через единый rAF-цикл
> [ScrollOrchestrator.tsx](web/src/components/ScrollOrchestrator.tsx). Никаких
> per-section `useEffect`, никаких сторонних animation-библиотек. Внутренние
> страницы подключают движение **через data-атрибуты** (см. «Движок»).

## Антипаттерн (чего избегать)

То, что вышло плоско на первой версии methodology:

- вся страница — одинаковые `.tcard` в `.trust-grid`, секция за секцией;
- каждый блок одной высоты, один фон, один ритм;
- нет полноэкранных моментов, нет движения, нет «дыхания» между блоками;
- текст появляется весь сразу, статикой.

Apple так не делает. Их страницы = **чередование масштабов и режимов**: огромный
тихий hero → полноэкранный statement → плотный блок фактов → галерея → pinned
демонстрация на несколько экранов → спокойный финал.

## Движок (data-атрибуты)

ScrollOrchestrator поднимает их на любой странице (no-op на главной):

| Атрибут | Что делает | CSS-выход |
|---|---|---|
| `data-reveal` | появление при входе во вьюпорт | класс `.is-in`; варианты `="fade\|up\|left\|right\|scale"`; задержка `style="--reveal-delay:120ms"` (стаггер) |
| `data-scrub` | прогресс прохода элемента через вьюпорт | пишет `--scrub` 0→1 (parallax, заполнение ring, скраб медиа) |
| `data-pin` + `data-pin-steps="N"` | высокий трек + липкая сцена внутри (`data-pin-stage`); шаги загораются по очереди | пишет `--pin` 0→1; на `data-pin-step` — `.is-active` (накопительно), `.is-current` (текущий), `--pin-step` (индекс) |

Базовые стили примитивов — в `globals.css` (секция «Generic scroll-FX
primitives»), reduced-motion обработан там же и в движке. Конкретную раскладку
секции (grid, sticky-высоту трека, медиа) страница задаёт своими классами под
обёрткой страницы (напр. `.methodology …`) на **канонических токенах**.

## Каталог блок-архетипов

Собирай страницу из РАЗНЫХ архетипов, не повторяя один. Минимум 5–6 разных на
страницу из 8 секций.

1. **Statement hero** — крупный тихий заголовок (`clamp(44px,7vw,96px)`),
   lens-акцент на ключевой фразе (`.grad-word`), короткий подзаголовок, один CTA.
   Много воздуха. Часто — тёмный (`.band.ink`) или светлый full-bleed.
2. **Full-bleed statement** — одна мысль на весь экран (`min-height:100svh`),
   большой текст по центру, `data-reveal`. Разделяет плотные блоки, даёт «вдох».
3. **Pinned multi-screen** — одна секция на несколько экранов: `data-pin` +
   липкая сцена, по мере скролла построчно/поэтапно появляется текст или
   загораются узлы (`data-pin-step`). Это «как Apple рассказывает про продукт».
   Идеально для процесса/конвейера, «как это работает», timeline.
4. **Editorial split** — текст слева, визуал/медиа справа (или наоборот),
   чередовать сторону между секциями. `data-reveal="left"` / `="right"`.
5. **Horizontal gallery** — горизонтальная лента карточек со scroll-snap
   (`overflow-x:auto; scroll-snap-type:x mandatory`) для наборов (роли судей,
   use cases). Не вертикальная сетка.
6. **Stat band / counters** — крупные числа (mono, tabular, lens-градиент),
   появляются стаггером; каждое число — с источником-подписью.
7. **Scrubbed visual** — элемент с `data-scrub`: parallax-сдвиг, заполнение
   confidence-ring по `--scrub`, прокрутка медиа.
8. **Bento overview** — разнокалиберные тайлы (как на главной), один hero-тайл +
   несколько равных, для обзора возможностей.
9. **Quiet CTA band** — финальный спокойный призыв на контрастной полосе
   (`.band.ink` / `.band.soft`) + pill-кнопка.

## Композиция и ритм

- **Чередуй режимы поверхности:** светлая → тёмная (`.band.ink`) → светлая.
  Минимум один тёмный statement на страницу — даёт «кино».
- **Чередуй масштаб:** full-bleed момент → плотный блок → снова воздух. Не три
  плотных блока подряд.
- **Чередуй раскладку:** editorial-split меняет сторону; gallery горизонтальна
  среди вертикальных секций.
- **Один акцент на экран** (правило design-system): lens-градиент на числах/
  ключевой фразе, не везде.
- **Max-ширина** контента 1180–1200px (`.wrap`/`--maxw`); full-bleed-секции
  тянутся на 100vw, но текст внутри держит max-ширину.
- **Spacing крупный:** секции дышат (`padding: clamp(64px,10vw,160px) 0`),
  Apple-страницы щедры на воздух.

## Типографика в деле

- Hero/statement — `clamp(44px,7vw,96px)`, `line-height:1.02`,
  `letter-spacing:-.03em` (как `.hero .title` / epic-заголовки главной).
- Секционный заголовок — `clamp(33px,5vw,56px)`; на тёмном крупнее.
- Lens-акцент фразы — `.grad-word` (на внутренней странице нужен свой
  `<wrapper> .grad-word` правило с `--lens`, т.к. глобальный `.grad-word` scoped
  под `.hero`).
- Mono-лейблы (`.eyebrow`) — uppercase, трекинг `.16em`, ведущая `.dot`.

## Движение (motion)

- Easing — только `--ease` (замедление, без отскока).
- Reveal — `data-reveal` со стаггером внутри группы (`--reveal-delay` по индексу).
- Pinned-секции — `data-pin`; текст появляется по `.is-active`/`.is-current`.
- Parallax/scrub — `data-scrub` → `--scrub` в `transform`/`background`.
- **reduced-motion** — обязан быть тих: примитивы уже это уважают; кастомные
  эффекты оборачивай в `@media (prefers-reduced-motion:reduce)`.
- **Контент в pin-секциях гейти на `.is-active`, не на `.is-current`.** В
  reduced-motion движок выставляет только `.is-active` (всё показано) и НЕ
  `.is-current` (нет «активного» шага без скролла). Если текст раскрывается
  только по `.is-current` — добавь reduced-motion-оверрайд, иначе контент
  пропадёт у этих пользователей.

## Чек-лист «не плоско ли»

- [ ] ≥5 разных архетипов на 8 секций (не одна сетка карточек)
- [ ] ≥1 pinned multi-screen секция со scroll-driven появлением
- [ ] ≥1 тёмный full-bleed statement (смена режима)
- [ ] чередование светлых/тёмных полос и масштаба
- [ ] хотя бы одна горизонтальная галерея ИЛИ bento (не только вертикаль)
- [ ] движение через data-атрибуты (ScrollOrchestrator), не useEffect
- [ ] reduced-motion тих; только канонические токены/классы
