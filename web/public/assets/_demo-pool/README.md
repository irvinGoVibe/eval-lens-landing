# _demo-pool — библиотека стаб-медиа для component-forge

Сюда складываются **переиспользуемые demo-картинки/видео**, которыми блоки
наполняются вместо пустого `.media-ph` (статус — `demo-content`, см.
`.claude/skills/component-forge/kb/demo-media.md`). Цель — показывать блок с
**реальной картинкой**, а не с плейсхолдером.

## Как пользоваться

- Каталог с тегами — **[INDEX.md](INDEX.md)** (type/role/theme/ratio/text). Положил
  файл в `photo/` или `video/` — допиши строку в INDEX.
- Кидай сюда годные исходники (фото / скриншоты / короткие видео / постеры).
- `media-curator` (Фаза 4B) **читает INDEX.md и ищет здесь в первую очередь**, и
  только если не нашёл — идёт в ассеты архетипа → соседних блоков → во все ассеты проекта.
- Это **источник**: media-curator не правит файлы здесь, а делает производные
  (ресайз/пережатие/тема/desktop-mobile) в `web/public/assets/section-lab/<archetype>/`.

## Цвет под тему (важно)

Складывай так, чтобы было видно пригодность под тему:
- светлые/«белые» — для light-поверхностей;
- тёмные/«чёрные» — для dark-поверхностей;
- если ассет годится на обе — это `single-adaptive`.

## Нейминг (рекомендация, не жёстко)

`<тема>-<тема-сюжета>-<ориентир-размера>.<ext>`, например
`light-product-panel-wide.webp`, `dark-evidence-rows-square.jpg`,
`neutral-abstract-tall.webp`. Форматы — как в проекте (WebP/AVIF/PNG/JPEG; видео — mp4/webm).

> Папка может быть пустой на старте — наполняется по мере появления годных
> ассетов. Файлы здесь публично отдаются (`/assets/_demo-pool/...`).
