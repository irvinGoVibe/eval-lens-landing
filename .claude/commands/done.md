Ты — исполнитель команды `/done` из EvalLense Landing. Ведёшь дневной отчёт по работе. Полный контекст — `PROJECT-ENTRYPOINT.md` (секция «Daily end-of-day reports»), шаблон — `notes/done/_template.md`, классификация сигналов — `notes/README.md` → «Типы сигналов».

## Вход

`$ARGUMENTS` — опциональная короткая формулировка результата. Если пусто — работаешь в интерактивном режиме и спрашиваешь user, что зафиксировать.

## Что делаешь

1. **Определи сегодняшнюю дату** в формате `YYYY-MM-DD`.

2. **Проверь наличие файла** `notes/done/<YYYY-MM-DD>.md`:
   - если нет — создай по шаблону `notes/done/_template.md`, подставь сегодняшнюю дату в `title`, `date` и H1
   - если есть — открой существующий и **дополняй**, не переписывай

3. **Куда писать содержимое:**
   - если `$ARGUMENTS` короткое (одна строка результата) — добавь буллет в секцию **«Что сделано»**
   - если пусто или просит развёрнутого отчёта — собери факты:
     - прочитай `notes/daily/<YYYY-MM-DD>.md` (если есть) — это источник сырых сигналов за день
     - прочитай git-лог за сегодня (если работаешь в репо): `git log --since='00:00' --oneline --author=<user>`
     - прочитай изменённые файлы через `git status` / `git diff --stat`
     - на основе этого предложи пользователю черновик четырёх секций («Что сделано», «Что в процессе», «Что заблокировано», «Ссылки»)
   - в обоих случаях **покажи diff** до записи и жди подтверждения

4. **После подтверждения — запиши.** Сохраняй правило append-only: не переписывай строки, добавленные ранее сегодня или в прошлые дни.

5. **Отчёт user.** После записи верни:
   - путь к файлу
   - что добавил (суть изменений, кратко)

## Правила

- **Один файл на день.** Не создавай несколько файлов на одну дату.
- **Прошлые дни не трогаешь.** Файлы `notes/done/<прошлая-дата>.md` append-only в принципе — никакие правки задним числом.
- **Не дублируй `notes/daily/`.** `daily/` — сырой поток, `done/` — структурированный итог. Если то же самое уже в daily, в done пиши обобщением, а не копипастой.
- **Никаких Linear Issues.** `/done` не публикует ничего во внешние системы. Это личный worklog.
- **Работает по шаблону из `notes/done/_template.md`.** Если шаблон изменился — следуй новой версии.

## Чего не делаешь

- не редактируешь файлы прошлых дней
- не удаляешь содержимое сегодняшнего файла при повторном вызове
- не создаёшь Linear Issue, PR или коммит — только запись в `notes/done/`
- не пишешь в `daily/` из этой команды — для этого есть `/note`
- не ставишь сегодняшнему отчёту статус «final» или что-то подобное — дневной отчёт принципиально не имеет состояния
