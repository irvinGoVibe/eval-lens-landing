# Rule: task-packets (что передаём агенту — не весь чат)

**Источник правды:** оркестратор `component-forge/SKILL.md`, ростер `workers.md`.

Оркестратор **не передаёт агенту весь диалог**. Каждый агент получает короткий
пакет — только нужное для его фазы:

```
- archetype ID
- current phase + текущее состояние (process-states)
- exact role (что именно делает этот агент)
- approved inputs (контракт, карта, DS-ограничения, утверждённые брифы/медиа)
- relevant files (конкретные пути, не «весь репо»)
- design-system constraints (+ surface-to-action матрица)
- media brief (если фаза про медиа)
- allowed files (что можно трогать)
- forbidden actions (что нельзя — см. рейлы оркестратора)
- expected output (формат результата)
- stop condition (когда остановиться / что вернуть)
```

## Правила

- Оркестратор **собирает результаты** агента и передаёт дальше **только
  необходимые данные**, а не сырой вывод целиком.
- Нужные rule-cards из `kb/` вставляются в пакет **дословно** (агент-дженерик так
  получает проектные правила).
- Один frontend writer на архетип; read-only reviewers — параллельно.
- Пакет ссылается на плейбук фазы (`forge-*/SKILL.md`), но не заменяет его пересказом.
