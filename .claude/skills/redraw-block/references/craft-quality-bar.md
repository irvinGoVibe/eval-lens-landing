# Craft quality bar — планка продуктовой вёрстки

Задача скилла: **взять готовый структурный блок (скелет: какие секции, порядок,
слоты контента) и поднять его вёрстку до современного продуктового качества** —
не меняя структуру, а доводя **исполнение**.

Грамматика (`apple-block-grammar.md`) отвечает на «**что** это за блок» —
композиция, иерархия, ритм. Этот документ — про «**насколько хорошо он сделан**»:
финиш, детали, состояния, отзывчивость. Структура переиспользуется; апгрейдится
качество.

> Тест продуктового уровня: блок выглядит так, будто его довёл senior
> product-дизайнер + senior фронтендер — без «почти ровно», «и так сойдёт»,
> дефолтных состояний и случайных значений.

---

## 10 осей качества

### 1. Spacing & вертикальный ритм
- Отступы — из единой шкалы, не «на глаз». Соседние блоки и элементы внутри
  подчиняются одному ритму; нет случайных `13px`/`27px`.
- Воздух щедрый, но дисциплинированный (грамматика, правило 3). Внутренние
  группы сгруппированы близостью (label ближе к своему значению, чем к соседнему).
- Контент в `--maxw` / `--gutter`; текстовая колонка не во всю ширину экрана.

### 2. Типографика — детали
- Размеры — `clamp()` (флюидно), без скачков на промежуточных ширинах.
- `line-height` под роль: плотный для крупных `.title`, просторный для body.
- Display-заголовки — выверенный `letter-spacing` (обычно отрицательный на крупном).
- `text-wrap: balance` для заголовков, `pretty` для абзацев — нет висячих слов
  и кривых переносов.
- Никаких сирот/одиночных слов на строке в заголовке; читаемая длина строки body.

### 3. Выравнивание
- Оптическое выравнивание, не только математическое; общие кромки у соседних
  элементов; единая левая граница колонки.
- Иконки/числа/текст выровнены по базовой линии, а не «примерно».
- Сетки — реальная grid/flex-раскладка с равными промежутками, не магические марджины.

### 4. Цвет и поверхность — только токены
- Цвета/фон/текст — исключительно из токенов (`--bg`/`--bg-soft`/`--ink`/`--fg`/
  `--fg-on-dark`/lens-акценты). Ни одного хардкод-hex.
- Пара текст↔фон корректна для поверхности (на ink — `--fg-on-dark`/`--body-on-dark`).
- Контраст ≥ WCAG AA для body; крупный текст — ≥ AA Large.
- Тонкие линии — из `--border` / `--border-on-dark`, не произвольный серый.

### 5. Радиусы и тени
- Радиусы — из `--r-xs…--r-pill`, согласованно (не смесь `8px`/`10px`/`12px`).
- Тени — из `--sh-1…--sh-3` (фиолетовые, не серые «дефолтные»); elevation осмыслен,
  не у каждого элемента подряд.

### 6. Интерактивные состояния (обязательно все)
- `:hover`, `:focus-visible`, `:active`, `:disabled` — **спроектированы**, не
  дефолтные браузерные. Особенно `:focus-visible` — видимый доступный фокус-ринг.
- Переходы состояний плавные (`transition` на нужных свойствах, easing `--ease`),
  без резких скачков.
- Кликабельное выглядит кликабельным; не-кликабельное — нет.

### 7. Финиш движения
- Анимируем только `transform`/`opacity` (перф; без layout-thrash).
- Движение — через `data-*` в `ScrollOrchestrator`, easing `--ease`, сдержанно.
- `prefers-reduced-motion` — корректная деградация (контент доступен без анимации).
- Никакого CLS: ассеты резервируют место (`aspect-ratio`/`--ratio`), ничего не
  «прыгает» при загрузке.

### 8. Отзывчивость
- Флюидно от десктопа до мобайла; контент **переукладывается**, а не просто
  ужимается; нет горизонтального оверфлоу.
- Тач-таргеты ≥ 44px; интерактив удобен на мобиле.
- Длинные/пиннящиеся блоки (грамматика, правило 9) деградируют в обычный
  вертикальный reveal на узких/`reduced-motion`.

### 9. Детали и полировка
- Кромки чёткие; нет рассинхрона в 1px и «дрожащих» границ.
- Изображения — корректные `alt`, зарезервированный `aspect-ratio`; нет растяжения.
- Нет пустот без смысла, обрезанного текста, наезжающих элементов на краях.
- Состояния пусто/загрузка/ошибка (если применимо) — не «голый» дефолт.

### 10. Качество разметки
- Семантический HTML (заголовки/списки/секции по смыслу), доступные роли/aria
  там, где нужно.
- Секция остаётся Server Component, если не нужен интерактив; `"use client"` не
  ради удобства.
- Классы — токен-ориентированные (legacy-классы globals.css или Tailwind-утилиты);
  нет мёртвых стилей и дублей.
- Никакого хардкода значений, которые есть в токенах.

---

## Чего НЕ делаем
- Не меняем структуру блока (порядок секций, слоты, что показываем) — это вход,
  а не предмет редизайна. Меняем структуру только по явной просьбе user'а.
- Не трогаем контракт компонента (пропсы/API/серверность/данные) — правило 10
  грамматики.
- Не «украшаем» декором (лишние градиенты/тени/свечения) — продуктовое качество
  это про точность и финиш, а не про добавление эффектов.

## Чек продуктового качества
- [ ] Структура блока сохранена; апгрейднут только финиш вёрстки.
- [ ] Spacing/ритм из единой шкалы; контент в `--maxw`/`--gutter`; нет «на глаз».
- [ ] Тип: `clamp()`, выверенные line-height/tracking, `text-wrap` balance/pretty, нет сирот.
- [ ] Выравнивание оптическое и по базовой линии; равные промежутки в сетках.
- [ ] Цвет/радиус/тень — только токены; пара текст↔фон корректна; контраст ≥ AA.
- [ ] Все состояния (hover/focus-visible/active/disabled) спроектированы; фокус виден.
- [ ] Движение: transform/opacity, `--ease`, через `ScrollOrchestrator`, reduced-motion ок, нет CLS.
- [ ] Отзывчиво: переукладка, нет оверфлоу, тач-таргеты ≥44px.
- [ ] Семантичный HTML; Server Component где можно; нет мёртвых стилей/хардкода.
