# page-validate — аудит страницы против `<slug>_new.md`

Берём ОДНУ уже собранную страницу и проверяем её на двух уровнях:

1. **Live render QA** — переиспользуем дисциплину skill **`forge-validate`**, но
   применённую ко **всей странице**, а не к одному блоку.
2. **Doc-sync** — статически сверяем `page.tsx` с **новым** продуктовым брифом
   `wiki/product/<slug>_new.md` (именно `_new` — это свежие, утверждённые тексты).

Результат — **одна сводная таблица в чат** по секциям. Скилл **ничего не правит**:
только читает, прогоняет и докладывает. Одна страница за вызов.

> **Канон.** Публичный API секций — барель `@/components/ds`. «Актуальный»
> компонент = импортирован из `@/components/ds` (даже если внутри это re-export
> `Lab*` — это нормально, барель и есть канон). «Неактуальный» = page-local /
> инлайн-разметка, сырой `Lab*`/`_kit` мимо бареля. Контейнер страницы обязан
> нести `section-lab ds`, а `<ScrollFX/>` должен быть смонтирован — иначе
> DS-секции невидимы (см. CLAUDE.md).

## Фаза 0 — резолв страницы

Из аргумента (route или slug) определить:

- **slug** = последний сегмент пути (`/product/evidence-based-reports` → `evidence-based-reports`).
- **page.tsx** = `web/src/app/<route>/page.tsx`.
- **бриф** = `wiki/product/<slug>_new.md`.

Если `<slug>_new.md` **не найден** — остановиться и сказать об этом: новых текстов
для сверки нет, аудит текстов невозможен (можно предложить сверку с базовым
`<slug>.md`, но только по явному согласию — это НЕ те тексты, что просили).

Прочитать **оба файла целиком**. Канонический список секций и ожидаемые
архетип/движение/поверхность берём из таблицы **«Структура секций»** брифа;
финальную копию — из раздела **«Контент по секциям»**.

## Фаза 1 — live render QA (через `forge-validate`)

Применить чек-лист `forge-validate` ко **всей странице**:

- `cd web && pnpm build` зелёный;
- поднять дев-сервер (**порт 3005**, `preview_start` name `web`) и пройти браузером:
  консоль чистая (`preview_console_logs`); **нет page-level horizontal overflow**
  (`documentElement.scrollWidth <= clientWidth + 1`); намеренные scroll-snap рельсы
  (`Gallery`/`.seg-lane`) скроллятся внутри контейнера, не двигают страницу;
- **light/dark** парность геометрии; **responsive** на 375/768/1024/1280/1440;
- **motion**: reveal/pin/`tr-tone-flip` отрабатывают, анимация завершается до ухода
  элемента из вида, `prefers-reduced-motion` уважается; снять `preview_screenshot`
  desktop+mobile для отчёта.

> **Авторизация сервера.** Как и `forge-validate`, page-validate **авторизован**
> поднимать порт 3005 на время проверки — сам вызов `/page-validate` и есть тот
> «явный запрос», которого требует CLAUDE.md. Reviewers/проверки — read-only,
> страницу НЕ редактируем.

Если живой прогон по какой-то причине невозможен (или user просит только
текст-сверку) — выполнить Фазы 2–3 и пометить live-слой как `skipped`.

## Фаза 2 — посекционный doc-sync (статически)

Идём по каноническому списку секций брифа. Для **каждой** секции находим её
аналог в `page.tsx` и оцениваем по осям:

### A. DS-компонент актуален?
Маппинг «архетип из брифа → ожидаемый барель-компонент»:

| Архетип (бриф) | Барель `@/components/ds` |
|---|---|
| statement-hero | `StatementHero` |
| full-bleed-statement | `FullStatement` |
| pinned-multi-screen | `PinnedSteps` |
| horizontal-gallery | `Gallery` |
| editorial-split | `EditorialSplit` |
| bento | `Bento` |
| numbered / steps | `Numbered` |
| chip-grid | `ChipGrid` |
| stat-band | `StatBand` |
| quiet-cta | `QuietCta` |
| cinema / final-cta (видео) | `Cinema` |
| cta-band | `CtaBand` |
| hub-map | `HubMap` · risk-control → `RiskControl` · faq → `Faq` |

- ✅ — секция отрендерена барель-компонентом, и он соответствует архетипу брифа.
- ⚠️ — барель-компонент, но **архетип расходится** с брифом (напр. бриф `quiet-cta`,
  в коде `Cinema`) — расхождение типа, зафиксировать.
- ❌ — секция собрана **page-local/инлайн-разметкой** или сырым `Lab*`/`_kit` мимо
  бареля (напр. stat-band руками как `.evr-statband`, а не `StatBand`).

### B. Кастом-настройки?
Перечислить нестандартное на секции: `version`-оверрайд, `bare`, `columns`,
кастомный `slot` (вложенный компонент), page-local обёртки/секции, ручную
разметку рядом с барель-компонентом, любые пропсы вне «дефолтного» контракта.
«Нет» — если только контентные пропсы (eyebrow/title/sub/items) по контракту.

### C. Поверхность (бриф ↔ код)
Колонка «Поверхность» брифа vs проп `surface` в коде. Расхождение в ритме
light↔ink — ⚠️/❌ (нарушает анти-полосатость).

### D. Переход → следующая секция
Что стоит **между** этой и следующей секцией в коде:
- границы зоны `ds-zone` (open/close, общий фон `--lobes`/`--lobes-dark`);
- `tr-tone-flip--bloom` (`data-from`/`data-to`, `data-pin`) — флип light↔ink;
- прямой стык (без переходного блока).
Сверить, что тональные стыки получили переход (а не голый стык), и что
`data-from`/`data-to` совпадают с фактическими поверхностями соседей.

### E. Текст ↔ `_new.md`
Сверить eyebrow / heading (lead+accent+trail) / subhead / тексты карточек и шагов
в коде с разделом «Контент по секциям» брифа:
- ✅ — дословно (с поправкой на типографские апострофы `’`/`'` и тире);
- ⚠️ — мелкий дрейф (пунктуация, перестановка, лёгкая правка формулировки);
- ❌ — другой текст / отсутствует / лишнее, не предусмотренное брифом.

Секции/блоки в коде, которых **нет** в «Структуре секций» брифа (напр. page-local
hero stat-row), выносим отдельными строками с пометкой `page-local · не в брифе`.

## Фаза 3 — сводная таблица в чат

Вывести **одну** markdown-таблицу, строка = секция, в порядке брифа:

| # | Секция | DS-компонент (актуален?) | Кастом | Поверхность бриф↔код | Переход → след. | Текст ↔ `_new` |
|---|--------|--------------------------|--------|----------------------|-----------------|----------------|

Ячейки — `✅ / ⚠️ / ❌` + короткая нота в одну строку (что именно).

Под таблицей:

- **Детали расхождений** — по каждому ⚠️/❌: что в брифе, что в коде (для текста —
  обе строки; для архетипа/поверхности — ожидание vs факт).
- **Live render** — статусы из Фазы 1 (build / console / overflow / light·dark /
  responsive / motion) + ссылки на скриншоты, либо `skipped`.
- **Вердикт** — `clean` (всё ✅) / `drift` (есть ⚠️) / `out-of-sync` (есть ❌),
  и краткий список «что поправить», БЕЗ внесения правок.

> page-validate не чинит и не коммитит. Исправления — отдельным проходом
> (`redraw-block` для качества секции, `retype-section` для смены архетипа,
> ручная правка копии). Скилл лишь даёт точную карту расхождений.

## Definition of done
- [ ] Резолвлены `page.tsx` и `<slug>_new.md`; если `_new` нет — честно остановлено.
- [ ] Live render по `forge-validate` пройден (или явно `skipped`), скриншоты сняты.
- [ ] Каждая секция брифа оценена по осям A–E; page-local блоки вынесены отдельно.
- [ ] Одна сводная таблица + детали расхождений + вердикт выведены в чат.
- [ ] Ни одного изменения в коде/доках; правки только предложены.
