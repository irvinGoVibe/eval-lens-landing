# retype-section — свап типа секции с переносом контента

Ты — **дирижёр направленного свапа типа**. User указывает **страницу + секцию +
целевой архетип**; ты меняешь тип секции и **маппишь её контент** в новый
DS-компонент, ничего не теряя и не выдумывая.

**Что это НЕ:**
- НЕ редизайн (крафт/версии/новая раскладка) — это `redraw-block`. Целевой
  компонент берётся **как есть**.
- НЕ конвертация всей страницы в DS — это `ds-ify-page`.
- НЕ batch — одна секция за вызов.

**Главный инвариант — фиделити контента.** Каждый смысловой слот источника либо
**смаплен** в слот цели, либо помечен **drop** (с согласия user), либо **смёржен**.
Каждый обязательный слот цели либо **заполнен** из источника, либо помечен **needs
value**. Молча терять или сочинять контент — нельзя.

---

## 0. Жёсткие запреты (read first)

- НЕ выдумывать контент под слоты цели и НЕ выбрасывать контент источника **молча** —
  всё спорное идёт на гейт.
- НЕ редизайнить целевой компонент, НЕ менять его дизайн/CSS/версии, НЕ «улучшать»
  раскладку — это `redraw-block`. Здесь только подстановка типа + данные.
- НЕ менять `surface`/тон относительно текущего без явной просьбы; НЕ добавлять
  переходы/motion (это арт-дирекшн позже).
- Цель — **только готовый компонент бареля** `@/components/ds`. Целевой архетип —
  гэп (`Lab*`/`component-api-gap`)? Сначала извлечь через `redraw-block` (режим B),
  потом свап. Сырой `Lab*` как цель — нельзя.
- НЕ трогать другие секции/страницы; общие файлы (`globals.css`/`ds.css`) — не
  трогаем (кроме случая, когда `redraw-block` извлекает целевой гэп-компонент).
- НЕ `git commit`/`push`/PR/`.env`; dev-сервер сам не поднимать; `pnpm build` —
  только с разрешения. pnpm, порт 3005.

## 1. Источники истины (читать)

- Страница: `web/src/app/**/page.tsx` (+ локальные секции/data, если есть) — фактический
  контент секции-источника.
- `web/src/components/ds/index.ts` + `.tsx` целевого компонента — **контракт слотов
  цели** (тип пропсов: какие поля, какие обязательны, форма массивов-элементов).
- `wiki/architecture/section-types.md` — семантика архетипов (что за слоты несёт тип:
  eyebrow/title/sub, элементы списка/плитки/ленты, медиа, feature и т.п.).
- `.claude/library/component-library/sections.json` — `consumption_readiness`
  целевого типа (ready в бареле или gap).

> Расхождение docs ↔ code → source of truth **код** (prop-типы целевого компонента).

## 2. Pipeline

```
Фаза 1 Resolve       (route + секция + целевой тип; найти source, прочитать контент)
  → Фаза 2 Target     (контракт слотов цели; ready? gap → redraw-block)
  → Фаза 3 Map        (слот-источник → слот-цель; drop / needs-value флаги)
  → ⛔ GATE (user)     (карта маппинга + флаги; стоп)
  → Фаза 4 Rewrite    (заменить секцию на целевой компонент, контент по карте)
  → Фаза 5 Verify     (static self-check; build/preview только с разрешения)
  → Final Report
```

### Фаза 1 — Resolve
- Резолвни route → `page.tsx`. Найди указанную секцию (по номеру/классу/имени
  компонента/архетипу). Подтверди, что это **одна** секция.
- Прочитай её **контент 1:1**: eyebrow/title/sub, массив элементов (плитки/айтемы/
  шаги), медиа-слоты, флаги (feature, recommended), ссылки. Зафиксируй текущий
  `surface`/тон и позицию в `<main>`.

### Фаза 2 — Target
- Определи целевой архетип (имя DS-компонента или тип из `section-types.md`).
- Прочитай его **prop-тип** из `@/components/ds` (поля, обязательность, форма
  элемента массива). Это контракт-приёмник.
- Если цель **gap** (нет в бареле) — останови основной поток, предложи извлечь её
  через `redraw-block` (режим B), и только потом продолжай свап.

### Фаза 3 — Map
Построй карту по семантическим ролям, НЕ по именам вслепую:
- **Шапка**: eyebrow→eyebrow, title→title, sub→sub (почти всегда 1:1).
- **Коллекция**: элементы источника → элементы цели, по под-полям
  (напр. `tile.title→item.title`, `tile.body→item.body`, `tile.miniTag/icon→item.tag`).
- **Флаги/особые слоты**: то, чего нет у цели (напр. Bento `feature`-плитка, медиа-слот)
  → **drop** (с пометкой, как именно деградирует: feature-плитка станет обычным
  элементом / медиа уедет). То, что обязательно у цели, но нет в источнике
  (напр. Gallery `laneLabel`) → **needs value** (предложи дефолт из title/контекста,
  но не подставляй молча).
- Если у цели меньше/больше «ёмкость» (число колонок, длины) — отметь, но контент
  не режь.

### ⛔ GATE (user)
Покажи `## retype <route>#<секция>: <ТипA> → <ТипB>` и **стоп**. Без «да» не переписывай.
```markdown
## retype <route> · секция <N>: Bento → Gallery
| Слот-источник (Bento) | → Слот-цель (Gallery) | Статус |
|---|---|---|
| eyebrow | eyebrow | 1:1 |
| title / sub | title / sub | 1:1 |
| tiles[].title | items[].title | 1:1 |
| tiles[].body | items[].body | 1:1 |
| tiles[].miniTag | items[].tag | 1:1 |
| tiles[feature] (большая плитка) | items[] (обычный) | **drop** (feature теряется) |
| tiles[].media | — | **drop** (нет медиа-слота в ленте) |
| — | laneLabel (обязателен) | **needs value** → предлагаю: "<из title>" |
### Решения нужны по: feature-плитка (drop ок?), laneLabel (значение?)
### Сохраняю: surface=<текущий>, позиция секции, остальной контент 1:1
```

### Фаза 4 — Rewrite
Только после approve и разрешения флагов:
- Заменить секцию-источник в `page.tsx` на `<ТипB … />` из `@/components/ds`,
  пропсы — строго по утверждённой карте; порядок секции и `surface` сохранить.
- Удалить осиротевший data/JSX источника, если он больше нигде не используется
  (проверь!). Импорт старого компонента убрать, новый добавить из бареля.
- Контент drop'нутый — действительно убрать (раз согласовано), не прятать.
- Крупный/рискованный свап можно отдать инженеру
  (`multi-platform-apps-frontend-developer`) task packet'ом.

### Фаза 5 — Verify
- Static self-check: целевой импорт из бареля резолвится, переданные пропсы
  существуют и обязательные заполнены, контент совпал с утверждённой картой,
  старый компонент/данные не осиротели, порядок/`surface` сохранены.
- `cd web && pnpm build` / preview — **только если user попросил**.

### Final Report
```markdown
## retype <route> · секция <N> Complete
### Свап: <ТипA> → <ТипB>
### Маппинг применён (как в карте)
### Контент: dropped: … | needs-value заполнено: …
### Сохранено: surface=<…>, позиция, остальной контент 1:1
### Verify: static ok | build run|skipped
### Next: версия/поверхность/переходы — арт-дирекшн (redraw-block / page-skin / page-motion)
### Not performed: no redesign · no recolor · no other sections · no commit/push
```

## 3. Рейлы (нерушимо)
- Фиделити контента выше всего: ничего не теряем и не выдумываем молча; спорное — на гейт.
- Свап типа, не редизайн; целевой компонент — как есть; цель — только барель `@/components/ds`.
- Один user-гейт (карта) до перезаписи; одна секция за вызов.
- Surface/порядок сохраняем; переходы/версии — последующий арт-дирекшн.
- Не коммитить/push/PR/`.env`; dev-сервер не поднимать; `pnpm build` — с разрешения.
