# zone-background — общий фон под группу идущих подряд секций

Ты — **дирижёр тональной зоны**. User указывает **страницу + диапазон подряд
идущих секций + пресет тона**; ты оборачиваешь эту группу в **одну зону**
(`.ds-zone`) с **единым фоновым слоем**, а сами секции переводишь на прозрачность,
сохраняя их контент, порядок и тон.

**Ключевая модель (повтори про себя):** фон — это **НЕ проп секции**. Это
отдельный слой `.ds-zone__bg` под **группой**. Зона делает все вложенные `.band`
прозрачными; секция через `surface` задаёт **только тон текста** (light/ink), а не
фон. Поэтому «один фон под пятью секциями» = одна зона + один bg-слот.

**Что это НЕ:**
- НЕ редизайн секций (крафт/раскладка/версии) — это `redraw-block`.
- НЕ переходы МЕЖДУ зонами/секциями (tr-* полки) — это `page-motion`.
- НЕ ковка новых фоновых вариантов — это `visual-layer-forge`. Здесь —
  применение **готовых** пресетов.
- НЕ batch — одна зона за вызов.

## 0. Жёсткие запреты (read first)

- НЕ менять контент, порядок и число секций. Только обёртка + класс зоны + bg-слот
  + правка `surface` под тон.
- НЕ изобретать новый фоновый CSS. Только готовые классы из `ds.css`:
  `--lobes` (light), `--lobes-dark` (dark), `ds-zone--flip` (flip). Нужного пресета
  нет → останов, маршрут в `visual-layer-forge` (не лепить локально).
- НЕ затаскивать в зону секции со СВОИМ full-bleed фоном, который зона НЕ
  подавляет: **Hero** (`lab-hero__bg`), **Cinema** (`ds-cinema`/видео), **Footer**
  (`variant="dark"`). Они остаются снаружи (full-bleed акценты). HubMap / Bento /
  Gallery — **можно** внутрь: их balls-glass бэкдроп зона гасит автоматически
  (`ds.css` :is(.ds-zone,.ds-canvas) …::after{background:none}).
- НЕ ставить переходы/motion и НЕ перекрашивать бренд-цвет секций.
- НЕ `git commit`/`push`/PR/`.env`; dev-сервер сам не поднимать; `pnpm build` —
  только с разрешения. pnpm, порт 3005.

## 2. Пресеты фона (выбор тона зоны)

> bg-слоты ниже даны для **частичной** (mid-page) зоны — со слотом-модификатором
> `ds-zone__bg--contained` (обязателен, см. заметку про fixed/absolute). Для зоны на
> ВСЮ страницу `--contained` **убери** (тогда fixed-слой даёт лёгкий параллакс).

| Пресет | Когда | bg-слот(ы) внутри зоны | surface секций |
|---|---|---|---|
| **light** | вся группа светлая, тёплый фон с переливами-блобами | `<div class="ds-zone__bg ds-zone__bg--contained ds-canvas__bg--lobes" aria-hidden />` | все `light` |
| **dark** | вся группа тёмная, «вспышки» на чёрном | `<div class="ds-zone__bg ds-zone__bg--contained ds-canvas__bg--lobes-dark ds-zone__bg--on" aria-hidden>` + 3 `__spark` | все `ink` |
| **flip** | группа должна перейти light→dark по скроллу | два `--flip-light` + `--flip-dark is-dark` (см. ds.css) + `[data-pin]`-драйвер на стыке; flip обычно идёт на всю страницу → fixed | до стыка `light`, после — `ink` |

Заметки:
- **Зона на ВСЮ страницу vs ПОДМНОЖЕСТВО секций — критично для фона.**
  `--lobes` / `--lobes-dark` по умолчанию `position:fixed`. Это верно ТОЛЬКО для
  зоны на всю страницу (нет секций-соседей вне зоны, как `/dev/canvas-bg`). Для
  **частичной** mid-page зоны fixed-слой накрывает весь вьюпорт и — из-за
  `isolation:isolate` на `.ds-zone`, который поднимает его НАД соседними секциями —
  **рисуется поверх секций выше/ниже зоны, и они пропадают** (особенно ink, у
  которых нет своего stacking-контекста — Hero может «выжить», ink-секции нет).
  → Для подмножества **ВСЕГДА добавляй `ds-zone__bg--contained`** (модификатор в
  ds.css: `position:absolute`) — он прибивает фон к боксу зоны (скроллится вместе
  с ней, заполняет только её регион). Проверяй геометрию слота: `position:absolute`
  и `getBoundingClientRect().top` ≈ верх зоны, НЕ накрывает вьюпорт на скролле 0.
- **dark** по умолчанию `opacity:0` (он спроектирован проявляться в flip-шве) —
  поэтому для статической тёмной зоны добавляем `ds-zone__bg--on` (модификатор уже
  есть в ds.css). Без него фон будет невидим.
- **Блобы/спарки** — опциональные энхансеры. Светлые плавающие PNG-блобы =
  `CanvasBlobs` (см. `dev/canvas-bg/CanvasBlobs.tsx`); тёмные «вспышки» = три
  `<span class="ds-canvas__spark ds-canvas__spark--N"/>` внутри dark-слота.
  Предлагай их, но не навязывай — базовый градиент работает без них.

## 3. Pipeline

```
Фаза 1 Resolve     (route → page.tsx; найти диапазон секций; проверить «подряд»)
  → Фаза 2 Triage   (для каждой секции: можно внутрь зоны? тон? own-backdrop → наружу)
  → Фаза 3 Plan     (пресет тона; разметка зоны; список surface-правок; проверка section-lab/ScrollFX)
  → ⛔ GATE (user)   (план зоны: внутри/снаружи + тон каждой; стоп)
  → Фаза 4 Apply    (обернуть группу в зону, добавить bg-слот, поправить surface)
  → Фаза 5 Verify   (static self-check; build/preview только с разрешения)
  → Final Report
```

### Фаза 1 — Resolve
- Резолвни route → `page.tsx`. Перечисли секции `<main>` по порядку (номер →
  компонент/архетип → текущий `surface`).
- Найди указанный диапазон. **Подтверди, что секции идут СТРОГО подряд** — между
  ними нет посторонней секции. Если между ними вклинивается own-backdrop секция
  (Hero/Cinema/Footer) → зона разрывается: вынеси это на гейт (либо две зоны, либо
  сузить диапазон).

### Фаза 2 — Triage
Для каждой секции диапазона реши:
- **own-backdrop** (Hero / Cinema / Footer / любой full-bleed, чей фон зона НЕ
  гасит) → **снаружи зоны**. Если она в середине диапазона — диапазон режется.
- HubMap / Bento / Gallery / Statement / Risk / Pinned и пр. контентные → **внутрь**.
- Текущий `surface` → целевой тон по пресету (light→`light`, dark→`ink`,
  flip→`light` до стыка / `ink` после).

### Фаза 3 — Plan
- Выбери пресет (light/dark/flip) — из явной просьбы user или предложи по текущим
  тонам секций.
- Определи **точку обёртки**: если зона = вся страница → класс на `<main>`; если
  подмножество → `<div className="ds-zone">…</div>` вокруг группы.
- Проверь инфраструктуру: на контейнере есть `section-lab` и `ds`? Смонтирован
  `<ScrollFX/>`? Чего нет — добавить (это обязательные условия: без них Lab*-стили
  схлопнутся, а flip/reveal не оживут).
- Для flip: где **стык** (последняя light-секция → первая ink) и какой
  `[data-pin]`-драйвер его двигает.

### ⛔ GATE (user)
Покажи `## zone <route>: секции <N–M> → <пресет>` и **стоп**. Без «да» не переписывай.
```markdown
## zone <route> · секции 3–6 → light (--lobes)
**Внутрь зоны (прозрачные, общий фон):**
| # | Секция | surface: было → станет |
|---|---|---|
| 3 | FullStatement | light → light |
| 4 | RiskControl   | soft → light |
| 5 | Gallery       | light → light |
| 6 | PinnedSteps   | light → light |
**Снаружи (свой full-bleed фон, не трогаю):** 1 Hero, 7 Cinema, 8 Footer
**Обёртка:** `<div className="ds-zone">` вокруг 3–6 · bg-слот: `--lobes`
**Инфра:** section-lab ✓ · ds ✓ · ScrollFX ✓ (или: добавлю)
**Не меняю:** контент, порядок, бренд-цвет, переходы
### Нужно решение по: пресет тона (light?), включать блобы/спарки?
```

### Фаза 4 — Apply
Только после approve:
- Обернуть группу: класс зоны на `<main>` **или** `<div className="ds-zone">…</div>`
  вокруг секций. Контейнер обязан нести `section-lab ds`.
- Добавить **первым ребёнком зоны** bg-слот(ы) по пресету (Фаза 2 таблица). Для
  dark — не забыть `ds-zone__bg--on`; для flip — два fixed-слоя + `[data-pin]` на
  стыке (разметка из ds.css-комментария).
- Поправить `surface` каждой внутренней секции под тон зоны. Контент/пропсы/порядок
  — без изменений.
- Если `<ScrollFX/>` не смонтирован — добавить в конец `<main>`. Импорт — из
  существующего места (см. другие страницы).
- Опциональные энхансеры (блобы/спарки) — только если user согласовал.

### Фаза 5 — Verify
- Static self-check: контейнер несёт `ds-zone` + `section-lab ds`; bg-слот первым
  ребёнком; внутренние секции получили нужный `surface`; own-backdrop секции
  остались снаружи; `<ScrollFX/>` присутствует; для dark есть `--on`, для flip есть
  драйвер. Контент и порядок не изменились.
- `cd web && pnpm build` / preview — **только если user попросил**.

### Final Report
```markdown
## zone <route> · секции <N–M> Complete
### Пресет: <light|dark|flip> (<fill-классы>)
### Внутри зоны: <список + surface> · Снаружи: <own-backdrop секции>
### Обёртка: <main | div.ds-zone> · Инфра: section-lab/ds/ScrollFX ok
### Энхансеры: блобы/спарки <added|skipped>
### Verify: static ok | build run|skipped
### Next: переходы между зонами/секциями — page-motion; крафт секций — redraw-block
### Not performed: no content/order change · no recolor · no transitions · no commit/push
```

## 4. Рейлы (нерушимо)
- Фон = слой под группой, не проп секции; `surface` = только тон. Контент/порядок —
  инвариант.
- Только готовые пресеты `ds.css` (`--lobes` / `--lobes-dark`+`--on` / `ds-zone--flip`);
  нужного нет → `visual-layer-forge`.
- own-backdrop секции (Hero/Cinema/Footer) — всегда снаружи; они рвут зону.
- Контейнер несёт `section-lab ds`; `<ScrollFX/>` смонтирован — обязательно.
- Один user-гейт (план зоны) до перезаписи; одна зона за вызов.
- Не коммитить/push/PR/`.env`; dev-сервер не поднимать; `pnpm build` — с разрешения.
