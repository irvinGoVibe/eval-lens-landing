# Component Library — machine-readable manifests

Generated by the `component-library-preparer` skill (full mode, 2026-06-17;
DS canon reframe 2026-06-22).
These are the **normalized consumption contracts** for the live EvalLense UI
library. Human-readable overview lives in
[`wiki/architecture/component-library.md`](../../../wiki/architecture/component-library.md).

## Who reads this
- **page-composer** — selects blocks. **Consumes ONLY entries with
  `consumption_readiness.status: "ready"`.** `conditional` requires an explicit
  fallback; `unavailable` is off-limits.
- **build-pages** — assembles internal pages from `ready` contracts.
- **component-forge** — picks up `component-api-gap` / inline items to forge.
- **forge-index** — calls the preparer in targeted incremental mode after
  registering a new archetype (updates one entry, not the whole library).
- **visual-layer-forge** — authors backgrounds (L13) / transitions (L14) / motion
  (L12) / composition recipes (L15) and registers each via the preparer in
  targeted incremental mode (`library_registration_request` task packet). A
  resource is not `ready` until `registration.status: completed`.

## Key facts (DS reframe 2026-06-22)
- **Public API is ONLY `@/components/ds`.** `Lab*`/`_kit`/`.lab-*`/`.section-lab`
  are a **deprecated internal substrate** the barrel wraps; direct import in
  page/skill code is forbidden. North-star: remove the substrate. See
  `manifest.json#consumption_api`.
- **Compose mode: `whole-sections-only`** — no DS layout shells yet (deprecated
  `_layout.tsx` also absent, CL-001), so page-composer composes whole DS sections.
- **5 sections are page-composer-`ready`** (in the barrel): StatementHero · Bento ·
  FullStatement · Gallery · PinnedSteps (browser QA carry-forward; geometry/API
  unchanged 2026-06-22; build = dev-compiled).
- **11 forged Lab* archetypes are `component-api-gap`** (consumption_readiness =
  `unavailable`) — not exported via `@/components/ds`; `component-forge` extracts
  each into clean DS before page-composer can use it. They remain forged at the
  forge level. (02,04,05,08,09,10,11,12,13,14,16)
- **Archetype 15 (Versus) is `unavailable`** — still inline JSX in the catalog,
  must be forged into a clean DS `Versus` before use.
- **Atom gaps:** Card/Chip/Stat/Ring/Bars/ListItem markup-only → `primitive-layer-forge`.
- **Dead code:** `FinalUnicorn`, `Trust`, `Results` (+ `UnicornStage`).
- **`variant="glass"`** must always render the shared liquid-glass material
  (selector group in `globals.css`) — see `atoms.json` Button constraint.

## Consumption API (added 2026-06-22)

The **only** public import surface is the prefix-free barrel `web/src/components/ds/index.ts`.
`Lab*` / `_kit` are a **deprecated internal substrate** the barrel wraps; direct
import of `Lab*`/`_kit` in page/skill code is **forbidden**. See
`manifest.json#consumption_api`.

```tsx
import { StatementHero, Bento, FullStatement, Gallery, PinnedSteps,
         Eyebrow, Title, Media, Button } from "@/components/ds";
```

- **Page container:** `<main className="section-lab ds">` — `ds` = public light
  design-system language (`--dsc-*` lens-soft cards, cool-mist atmosphere, deep
  media frame), gated on `:not(.ink)` so `surface="ink"` keeps the native dark
  look. `section-lab` = **temporary technical-debt** class, present only while
  `.lab-*` rules are still load-bearing; it goes away as `.lab-*` migrates into
  `ds.css`/`.ds`.
- **In the barrel (5 sections, page-composer-ready):** StatementHero · Bento ·
  FullStatement · Gallery · PinnedSteps. The other ~11 archetypes are **NOT in the
  barrel** — they are `component-api-gap`, extracted into clean DS by
  `component-forge`, never imported as `Lab*` directly.
- **Reference page:** `/dev/ds-sections` (composed entirely from the barrel);
  also `/dev/ds-atoms`, `/dev/ds-theme`.
- **No horizontal scroll:** enforced by `.band{overflow-x:clip}` (component-level
  guard) — `clip`, not `hidden`, so pinned `position:sticky` stages keep working.
- Full reference: `wiki/architecture/design-system.md` §Reusable components (v1.4).
  Cloud DS bundle (`.claude/designs/evallense` + claude.ai/design) synced 2026-06-22.

## Updating
- One thing changed → `/component-library-preparer "<id|name|path>"` (**targeted
  incremental**; accepts a background/transition/motion/recipe ID, a layer name,
  an export/source path, or an unambiguous description; rewrites only that entry +
  touched manifests). Several comma-separated targets run as independent updates.
- Only what git changed → `/component-library-preparer "changed"` (reads git diff,
  updates only affected library resources, handles rename/delete; no full scan).
- Full re-scan → `/component-library-preparer` (no args) or `… "full"`.
- A resource reaches `consumption_readiness: ready` only when
  `registration.status: completed` AND build evidence is honest (`dev-compiled` is
  not `passed`). Targeted/changed writes are draft-first (narrow audit → confirm),
  unless the user says "skip draft" / "publish immediately".
