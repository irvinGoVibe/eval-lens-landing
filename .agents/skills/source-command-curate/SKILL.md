# source-command-curate

Use this skill when the user asks to run the migrated source command `curate`.

## Command Template

Ты — исполнитель команды `/curate` из EvalLense Landing. Задача — перевести дозревший смысл из `notes/` в `wiki/` без потери информации и без нарушения стандартов. Полный контекст — `PROJECT-ENTRYPOINT.md`, правила работы с документами — `wiki/processes/process.md`.

## Что читаешь

- `notes/daily/*.md` за последние **14 дней**
- `notes/threads/*.md` полностью
- `notes/decisions/*.md` полностью

## Что делаешь

1. **Сгруппируй сырой материал по темам.** Найди повторяющиеся сигналы, устоявшиеся формулировки, решения, к которым команда вернулась более одного раза.

2. **Определи "дозревшие" темы.** Тема считается готовой к переносу, если выполнено хотя бы два условия:
   - упомянута в нескольких daily-файлах
   - есть конкретные примеры / acceptance / ограничения
   - терминология стабилизировалась
   - решение по теме принято

3. **Для каждой дозревшей темы предложи план переноса.** В отдельном markdown-отчёте пользователю покажи таблицу:

   | Тема | Куда в wiki/ | Имя файла | Status | Сырой источник |
   |---|---|---|---|---|

   - раздел wiki выбирается по смыслу: `wiki/product/` (продуктовые описания), `wiki/architecture/` (техническое устройство), `wiki/processes/` (рабочие процессы), `wiki/decisions/` (ADR — зафиксированные решения)
   - имя файла — lowercase, kebab-case, без дат и версий в имени
   - status всегда `generated` — это AI-создание; `review` / `approved` / `locked` ставит только человек

4. **Жди реакции пользователя.** Для каждой строки таблицы пользователь говорит: да / нет / позже / другой путь.

5. **Для каждой одобренной** — создай wiki-файл по шаблону `wiki/_template.md`:
   - frontmatter с `status: generated`, `version: 0.1`, `updated: <сегодня>` (AI-output)
   - back-link `← [[index|Wiki]]`
   - ссылки в коротком формате `[[name|Display]]`, если имя файла уникально в wiki
   - короткое вводное описание (что за документ, зачем)
   - перенос смысла из заметок — своими словами, не копипастой
   - в конец — секция «Связанное» с указанием исходных заметок через backtick-пути (например, `` `notes/daily/<YYYY-MM-DD>.md` ``), потому что cross-layer wiki-links запрещены

6. **Не трогай исходные заметки.** `notes/` остаётся append-only, даже после переноса смысла в wiki. Заметка — это исторический слой.

## Чего не делаешь

- не создаёшь документы со статусом выше `generated` (никогда `review` / `approved` / `locked`)
- не удаляешь и не редактируешь исходные заметки
- не пропускаешь проверку имени файла (lowercase, kebab-case)
- не используешь длинные ссылки `[[wiki/path/file|Display]]`, если имя файла уникально
- не переносишь сырую гипотезу, если не выполнены критерии дозревания

## Формат отчёта

Начни ответ с одной строкой: «Найдено N дозревших тем за 14 дней.» Дальше — таблица, затем для каждой темы короткий блок «Обоснование: почему дозрела». После — ждёшь команду пользователя.
