# Rule: version-protocol (v1 / v2 / v3 — лестница смелости)

**Источник правды:** redraw-block «Протокол версий».

## Правило

По умолчанию блок несёт **три версии возрастающей смелости** в `data-version="1|2|3"`.
Лестница по оси `structure`: **v1 ≈ слабо, v2 ≈ средне, v3 ≈ сильно**. **Каждая
версия — в ОБЕИХ темах (light И ink)** (одна геометрия, разный только цвет — см.
[surface-invariant](surface-invariant.md)). Контент-слоты держатся во всех версиях.

- **v1 — База (`structure: слабо`).** Раскладку почти не трогаем. Правильные
  цвета/шрифты/токены, корректная light/ink тема, surface-adaptive, чисто.
  Допустим лёгкий характер фона (мягкий градиент, hairline-grid) — сдержанно. Минимум риска.
- **v2 — Дорогой (`structure: средне`).** Сверяется с `interface-style-guide.html`
  и компонентами DS (status-чипы, confidence-ring, score, counters, window chrome).
  Поднимает композицию и финиш **внутри архетипа** — ритм, плотность, иерархия,
  состояния, выверенное движение. Цель — «дорогой» продуктовый вид.
- **v3 — Смелая (`structure: сильно`).** Сильно отличается: растяжение/сжатие,
  смена направления, 1→2 экрана (pin/scrub). Богатый визуал: видеофоны, фоновые
  картинки, жёсткие анимации/параллакс. Рейлы держим: движение через `ScrollFX`/
  `data-*` (см. [motion-wiring](motion-wiring.md)), медиа без ассета →
  `MediaPlaceholder`, glass — общая группа. light + ink **обязательно**
  (видеофон/оверлей перекрашивается под тему, геометрия не меняется).

## Механика

- Все три — в `data-version="N"`; неактивные `hidden`. Инспектор `LabMarkers`
  сам находит варианты, рисует табы v1/v2/v3 + тумблер Light/Dark.
- Версию **добавляешь, не удаляя** предыдущую — user сравнивает и оставляет лучшую
  (выбор в localStorage).
- Меньше/больше версий — только по явной просьбе user.

## Правка одной версии

Если задача — поправить конкретную версию по комментарию, правишь **именно её**,
новые версии не плодишь.
