# Rule: token-binding (только токены, без хардкода)

**Источник правды:** `web/src/app/globals.css` (`:root`), `.claude/designs/evallense/tokens/`.

## Правило

Все цвета/радиусы/тени/градиенты — **через CSS-переменные**. Никакого хардкода
hex/rgb/px-теней в новом коде. Новых цветов/теней/радиусов **вне `:root`** не вводим.

## Имена (из globals.css)

- Фон/текст: `--bg`, `--bg-soft`, `--ink`, `--fg`, on-dark — через `.band.ink`.
- Акценты: `--violet`, `--cyan`, `--aqua`, `--green`, `--amber`; lens-градиент `--lens` / класс `.grad-word`.
- Геометрия: радиусы `--r-*`, тени `--sh-*`, ширина `--maxw`, поля `--gutter`.
- Easing: `--ease` (Apple cubic-bezier).

## Обязательное

- ≤1 lens-акцент на блок (см. [copy-voice](copy-voice.md) и грамматику); остальное — нейтральная база.
- Захардкоженные значения в исходном инлайн-блоке — кандидат на чистку при извлечении.
- Заголовки — через `clamp()`; не фиксированные px-кегли.

## Проверка

`grep` по новому блоку: нет `#`-hex, нет `rgb(`/`rgba(` с литералами, нет
px-теней мимо `--sh-*`. Всё ссылается на переменные.
